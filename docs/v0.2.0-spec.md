## 1. 需求 brainstorm

代理是「无头」服务 —— 用户目前只能：
- 看 stdout 日志
- 翻 SQLite 数据库
- 跑 `curl /v1/health`

**这意味着用户调试时是「盲飞」**。Web 控制台的目标 = **可视化调试 + 实时监控**。

### 候选功能（影响 ROI 排序）

| # | 功能 | 用户价值 | 实现成本 | 建议 |
|---|------|---------|---------|------|
| A | **实时请求流**（SSE tail / 最近 100 条）| 🟢 极高 | 🟡 中 | ✅ **MUST** |
| B | **缓存命中率面板**（24h 折线 + 命中/未命中饼图）| 🟢 极高 | 🟢 低 | ✅ **MUST** |
| C | **配置热更新**（不重启改 yaml 部分字段）| 🟡 中 | 🟠 中高 | ⏸ v0.2.1 |
| D | **健康检查 + 上游连通性**（一键测试 MiniMax-M3）| 🟡 中 | 🟢 低 | ✅ MUST |
| E | **工具调用日志**（SSR tools 拦截 / 执行 / 回灌）| 🟢 高 | 🟡 中 | ✅ **MUST** |
| F | **Token 消耗统计**（按模型/时间/用户）| 🟡 中 | 🟠 中高 | ⏸ v0.2.1 |
| G | **代理设置面板**（改 yaml、持久化）| 🟡 中 | 🟠 中高 | ⏸ v0.2.1 |

**v0.2.0 范围 = A + B + D + E**（4 个面板，一次性推出）

## 2. 架构决策

### 决策 1：技术栈

| 选项 | 优势 | 劣势 |
|---|---|---|
| **FastAPI Jinja2 templates + htmx** | 零构建、Python 一把梭、SSR 友好 | UI 不够 fancy |
| FastAPI + React SPA | UI 灵活、组件化 | 需要 npm build 流程、复杂度↑ |
| FastAPI + Vue 3 CDN | 折中 | 双技术栈心智负担 |

**推荐**：**FastAPI Jinja2 + htmx + Alpine.js + Pico CSS**

理由：
- 14 个依赖已够多，**不加 npm 工具链**
- htmx 让 90% 的"局部刷新"用 5 行 HTML 搞定
- Pico CSS 1 个 CDN 链接搞定主题，不写 CSS
- 控制台是「运维/调试」场景，不是「产品」场景，**够用就好**

### 决策 2：数据流

```
[FastAPI 现有 /v1/* 端点] ── hook ──► [内存环形 buffer] ── poll 1s ──► [Web 控制台 SSE]
                                        (deque(maxlen=1000))
                                        
[Cache Store 已有 sqlite] ── query ──► [/admin/api/cache/stats] ──► [htmx 折线图]
                                        
[Settings 已加载] ── POST /admin/api/config/reload ──► [SIGHUP 给自己]
```

**关键约束**：
- **不影响现有 /v1/* 行为**（hook 写 buffer 用 try/except 包裹）
- **Web 控制台本身是只读**（v0.2.0 不做配置写）
- **认证**：v0.2.0 默认绑定 `127.0.0.1:8788`（**与代理不同端口**），**不暴露公网**
- 未来 v0.2.1 加 basic auth 时不影响 API

### 决策 3：端口与路径

| 路径 | 用途 |
|---|---|
| `:8787/v1/*` | 现有 Claude Code 兼容端点（**不变**） |
| `:8788/admin/` | **新增** Web 控制台（独立端口，独立进程？**嵌进现有 app**） |

**推荐**：控制台路由挂到现有 app（**不分离进程**），用 `127.0.0.1` 绑定防御。理由：少一个 systemd unit，少一份部署心智。

### 决策 4：可观测性数据源

| 数据 | 来源 | 采集成本 |
|---|---|---|
| 请求流 | 中间件 hook（`request_started` / `request_finished`）| 🟢 加 1 个 loguru sink |
| 缓存命中率 | `CacheStore` 已记录 hits/misses，加 1 个 counter | 🟢 现成 |
| 工具调用 | `ServerSideToolsHub.execute()` 已有 log，加 1 个 hook | 🟢 现成 |
| 上游连通性 | `UpstreamClient` 加 1 个 `ping()` 方法 | 🟢 现成 |

**所有数据都已存在或近似存在**，v0.2.0 主要是「**把它们暴露成 JSON / SSE**」。

## 4. 风险与回退

| 风险 | 缓解 |
|---|---|
| 控制台 hook 拖慢主流程 | 所有 hook 用 `try/except` 包裹，失败仅记 log |
| 内存 buffer 泄漏 | `deque(maxlen=1000)` 自动驱逐 |
| htmx 兼容性问题 | 备选：原生 fetch + 5 行 JS |
| 用户误开 `admin_enabled: true` 暴露公网 | 默认关 + README 红色警告 |
| 中间件 hook 顺序冲突 | 在 `main.py:188` 之前注册，独立 `include_router` |
